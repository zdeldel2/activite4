PARTIE 1 : Resumer de chapitre 1 (HIOUL + BENCHERIF)

Les techniques de l'information et de la communication (TIC) désignent l'ensemble des technologies utilisées pour collecter, traiter, transmettre et partager l'information. Elles englobent un large éventail d'outils et de méthodes, y compris les ordinateurs, l'internet, les réseaux de communication, les logiciels, les applications mobiles et bien plus encore.
Les TIC ont profondément transformé notre société et ont un impact majeur sur de nombreux aspects de notre vie quotidienne, tels que la communication, l'éducation, le commerce, la santé, les médias et le divertissement.
En termes de communication, les TIC permettent des échanges rapides et faciles d'informations à travers le monde. Les courriers électroniques, la messagerie instantanée, les appels vidéo et les réseaux sociaux sont autant de moyens de communication largement utilisés grâce aux TIC.
Dans le domaine de l'éducation, les TIC offrent de nouvelles opportunités d'apprentissage en permettant l'accès à des ressources éducatives en ligne, des plateformes d'apprentissage à distance et des outils de collaboration en ligne.
En ce qui concerne le commerce, les TIC ont favorisé le développement du commerce électronique et des transactions en ligne. Les consommateurs peuvent maintenant acheter des produits et des services en ligne, effectuer des paiements électroniques et suivre leurs commandes grâce aux TIC.
Dans le domaine de la santé, les TIC ont permis des avancées significatives. Les dossiers médicaux électroniques facilitent le stockage et le partage des informations médicales, les téléconsultations permettent aux patients de consulter des professionnels de la santé à distance, et les dispositifs de suivi de santé connectés permettent le suivi en temps réel des paramètres médicaux.
Les médias et le divertissement ont également été transformés par les TIC. Les services de streaming en ligne, les plateformes de partage de vidéos et les réseaux sociaux ont révolutionné la manière dont nous accédons aux contenus médiatiques et interagissons avec eux.
En résumé, les TIC sont un ensemble d'outils et de technologies qui ont un impact considérable sur notre société. Elles facilitent la communication, améliorent l'accès à l'information, favorisent l'apprentissage à distance, révolutionnent le commerce, transforment les soins de santé et modifient la façon dont nous consommons les médias et le divertissement

PARTIE 2 : Resumer de chapitre 2 (DJENKAL + AGGOUNE)

Ce chapitre porte sur les réseaux informatiques et leurs caractéristiques.
Un réseau est un ensemble d’ordinateurs et de périphériques autonomes connectés pour échanger des informations numériques selon des règles bien définies.
Les réseaux peuvent être classifiés en fonction de leur portée, qui peut être locale (LAN), métropolitaine (MAN) ou étendue (WAN).
Les caractéristiques de base d’un réseau incluent sa topologie, son débit, sa distance maximale et le nombre de nœuds maximum qui peuvent être interconnectés.
Les éléments d’un réseau comprennent les ordinateurs équipés de cartes de communication, les logiciels, les supports et les équipements d’interconnexion.
Les réseaux peuvent être homogènes ou hétérogènes en fonction des constructeurs des ordinateurs reliés.

PARTIE 3 : Resumer de chapitre 3 (MANSOURI + SENIA)

1)Les Composants matériel d’un Réseau informatique :
1 / Les cartes réseau : aussi appelées cartes d'interface réseau, sont des composants matériels qui permettent un ordinateur de se connecter un réseau, comme Internet ou un réseau local.
2/SWITCH :
Un switch est un composant matériel utilisé pour connecter des appareils sur un réseau local. Le switch permet plusieurs appareils, tels que des ordinateurs, des imprimantes, des serveurs, etc…De communiquer entre eux en utilisant des adresses MAC. Lorsqu'un appareil se connecte au switch, le switch vérifie son adresse MAC et stocke cette information dans une table de correspondance
3/HUB :
Un hub est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local.
il fonctionne comme un répéteur, il reçoit les données envoyées par un appareil et les transmet tous les autres appareils connectés.
2) Le Modèle OSI
OSI est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes.
• La couche physique : cette couche est responsable de la transmission des données sur le support physique.
• La couche liaison de données : cette couche est responsable de la transmission des données entre les appareils qui partagent un même support physique.
• La couche réseau : cette couche est responsable du routage des données à travers le réseau.
• La couche transport : cette couche est responsable de l'acheminement des données de bout en bout entre les appareils.
• La couche session : cette couche est responsable de l'établissement et de la gestion des sessions entre les appareils.
• La couche présentation : cette couche est responsable de la conversion des données dans un format qui peut être compris par l'appareil de destination.
• La couche application : cette couche est responsable de la communication entre les applications sur les appareils de réseau.
3) Le Modèle TCP/IP:
Le modèle TCP/IP est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes.
• La couche liaison de données : cette couche est responsable de la transmission des données sur le support physique (par exemple, câble, fibre optique, ondes radio).
• La couche réseau : cette couche est responsable du routage des données à travers le réseau.
• La couche transport : cette couche est responsable de l'acheminement des données de bout en bout entre les appareils.
• La couche application : cette couche est responsable de la communication entre les applications sur les appareils de réseau.
Le modèle TCP/IP est le modèle de référence pour les protocoles de communication de réseau les plus largement utilisés, tels que le protocole Internet, le protocole de contrôle de transmission et le protocole de datagramme utilisateur.
4) ADRESSAGE
Les communications dans un réseau obéissent à des règles :
• Adressage qui permet d'identifier de manière unique les deux unités en Communication.
• Architecture qui définit les rôles endossés par les deux unités .
• Protocoles qui assurent l'échange des données.

PARTIE 4 : Resumer de chapitre 4 (BENAMAR + BOURZAMA)

Internet est un support de communication général qui permet la transmission de données à travers des réseaux informatiques. Il s'agit d'un vaste réseau mondial qui connecte des millions d'ordinateurs et de dispositifs dans le monde entier.
Le World Wide Web (WWW), ou toile mondiale, est une application spécifique d'Internet. Il a été inventé par Tim Berners-Lee et Robert Cailliau.
Le Web est un système hypertexte public qui permet de consulter des pages web via un navigateur.
Les pages web sont accessibles sur des sites et sont reliées entre elles par des hyperliens, d'où l'image de la toile d'araignée.
Les URLs (Uniform Resource Locators) sont des adresses uniques qui permettent d'accéder à des ressources sur le Web. Elles contiennent l'adresse du serveur et l'adresse de la page sur le serveur.
Les aspects techniques du Web comprennent trois mécanismes de base
: les URLs pour localiser les ressources, le protocole HTTP pour la communication entre client et serveur, et le langage de description de pages web HTML qui permet de structurer le contenu des pages.
Une page web est un document multimédia contenant du texte, des images, des liens et d'autres éléments. Elle est décrite à l'aide du langage de description HTML et est interprétée et affichée par un navigateur. Les pages web sont regroupées en sites web, qui sont des ensembles de pages reliées entre elles par des liens hypertextes et partageant une thématique commune.
La création d'une page web HTML implique l'utilisation de balises HTML pour structurer le contenu du document. Un fichier HTML doit comporter au minimum les balises "<.html.>", "<.head>", "<.title> "et "<.body>"". Les titres sont définis avec les balises "<.h1>" à "<.h6>, et les paragraphes sont définis avec la balise "<.p>".

Git est un logiciel de gestion de version qui permet de conserver un historique des modifications effectuées sur un projet et de revenir à une ancienne version en cas de problème.
Les systèmes de gestion de version sont indispensables pour la gestion de projets et pour travailler en équipe de manière plus efficace.
Git est le leader incontesté parmi les logiciels de gestion de versions et est essentiel pour tout développeur.
les deux modèles des logiciels de gestion de version: le modèle centralisé et le modèle décentralisé (ou distribué).
Git est un exemple de modèle décentralisé qui présente des avantages tels que la simplicité et la flexibilité du travail, ainsi que la sécurité en permettant à chaque utilisateur de posséder une copie complète du projet en cas de corruption du serveur central.
GitHub est un service en ligne qui permet d'héberger des dépôts Git et est le plus grand hébergeur de dépôts Git du monde.
on explique aussi comment installer et configurer Git pour utiliser la version en ligne de commande.
Cloner un dépôt git :
1- créé un nouveau repo .
2- dans le CMD , écrire la commande suivante :
-git branch -M main
-git remote add origin
-https://github.com/BouchekirRedouane/ExmpleCloneGit.git
-git push -u origin main
Voilà, vous avez clone un dépôt git .
Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git :
Git permet de gérer les versions d'un projet en enregistrant les modifications dans un dépôt. Les commandes "git add" et "git commit" sont utilisées pour indiquer les changements à enregistrer, et il est important de faire un "git add" avant chaque "git commit" pour prendre en compte les dernières modifications.
Consulter l’historique des modifications Git:
Pour consulter les modifications , on utilise la commande "git log ".
Annuler des modifications apportées à un fichier : nous pouvons utiliser la commande générale "git checkout -- nom-du-fichier" ou la nouvelle commande spécialisée "git restore". Ces commandes nous permettent de restaurer un fichier à son état précédent enregistré dans Git.
Qu’est ce qu’une branche ? :
une branche dans Git est simplement un pointeur vers un commit spécifique, et la création d'une nouvelle branche crée simplement un nouveau pointeur plutôt que de recopier l'intégralité du répertoire de travail.
pour créer une branche en Git, on utilise la commande "git branch" suivie du nom de la branche. Ensuite, on utilise basculer sur la nouvelle branche avec la commande "git checkout".
